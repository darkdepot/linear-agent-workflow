## Backlog (vetted findings, not planned by default)

- **Installed-pack staleness visibility** (direction, MED): the install
  lockfile (`~/.codex/skills/.linear-agent-workflow.lock.json`) is written by
  the installer and read only by `install-local --check`; no workflow stage
  reports which pack version it is running, so a stale local install fails
  silently until behavior drifts. Cheap option: stamp
  `upstreamVersion`/`upstreamCommit` into the preflight certificate. Plan on
  request.
- **README/AGENTS/lifecycle rule duplication** (tech-debt, MED): lifecycle
  rules are restated in 3-4 places (e.g. delivery-ladder rules in
  `README.md:33-94`, `AGENTS.md:14-27`, `references/lifecycle.md`, and the
  skills). Real lockstep cost, but consolidation is delicate because
  `validate-workflow.mjs` pins exact sentences in README/AGENTS. Reassess
  after plan 004.
- **Validator string-coupling** (tests, by-design trade-off): ~80 exact
  sentences pinned in `scripts/validate-workflow.mjs` mean every intentional
  rewording requires a paired validator edit, while a same-meaning rewording
  silently weakens a guard. Accepted per the repo principle "scripts are only
  lightweight smoke guards" (`README.md:186`). Cheapest mitigation if it
  starts to hurt: a `references/validation-model.md` documenting which
  invariant each pin protects.

## Findings considered and rejected

(So nobody re-audits these.)

- **Null-dereference in `install-local.mjs check()`** (subagent
  CORRECTNESS-04): false — `scripts/install-local.mjs:309` guards the `else`
  branch; lock properties are only accessed when `lock` is non-null.
- **`artifactRoots`/`linearTeam` config fields unused** (subagent
  DIRECTION-02): false — `references/artifact-intake.md:15,24` and
  `skills/linear-handoff/SKILL.md:148` consume artifact roots; installed
  skills are instructed to read the config (incl. Linear team) via the note
  injected by `scripts/install-local.mjs:135`.
- **"No test harness for the scripts"** (subagent TESTS-02, as stated):
  overstated — `scripts/validate-workflow.mjs:259-391` functionally tests
  install and project-config in temp dirs, including negative fixtures. The
  *genuine* residual gaps (corrupt lockfile, --remove-stale) are covered by
  plan 002.
- **"Handoff-first ordering not validated"** (subagent TESTS-03): overstated —
  `validateAntiPatterns` pins the ownership chain (handoff must not own
  Delivery Start, implement owns it, ship requires the preflight certificate,
  deploy requires the green certificate; `scripts/validate-workflow.mjs:485-537`).
- **`artifactRoots` path-traversal validation** (subagent SECURITY-01): not a
  vulnerability — the config is authored by the repo owner for their own
  repo; no privilege boundary is crossed.
- **Unify the three certificate formats** (subagent DEBT-05): not worth doing —
  preflight/ship/deploy certificates carry stage-specific fields, are consumed
  pairwise by the next stage only, and are pinned by the validator; a unified
  schema adds migration churn with no consumer that needs it.
- **Add CONTRIBUTING.md** (subagent DX-04): not worth doing — `AGENTS.md` is
  this repo's contributor contract; a second document is a new drift surface.
  Fold any genuinely missing guidance into `AGENTS.md` instead.
- **Split `linear-handoff/SKILL.md` (261 lines)** (subagent DEBT-04): not now —
  the file is loaded on-demand as one unit and 261 lines is well within what
  an agent navigates reliably; splitting adds an indirection hop at install
  time. Revisit if it grows past ~400 lines.
- **Rewrite validators to semantic/behavioral checks** (subagent TESTS-01, as
  a fix): rejected as scoped — string pins are the repo's explicit design
  ("lightweight smoke guards"); behavioral simulation of agent skills is
  brittle and expensive. See backlog item on documenting the validation model.
- **Workflow status dashboard / fleet overview** (advisor idea during audit):
  rejected — Linear itself is the status surface for a single-operator
  workflow; certificates are already persisted as Linear comments.
