# Linear Idea

Use this wrapper to turn a raw idea into a strengthened Linear Project in Idea.

Read first:

1. `AGENTS.md`
2. `skills/linear-project/SKILL.md`
3. `skills/linear-check/SKILL.md`
4. `references/lifecycle.md`

Workflow:

1. Inspect available repo and Linear context before asking questions.
2. Ask 2-4 high-leverage AskQuestion-style questions.
3. Each question must include answer options, one recommended option, and a short reason for the recommendation.
4. Accept custom user answers when options do not fit.
5. Build a strengthened brief from the raw idea and answers.
6. Use `linear-project` to create or update Project in Idea.
7. Recommend `/brainstorming` or `/office-hours` and explain why.
8. Run or report `linear-check idea`.

Rules:

- Do not create PRD, Tech Spec, or Issue during Idea.
- Do not start implementation.
- Do not treat the raw idea as shaped requirements.
- Keep Linear-facing output in Russian.
