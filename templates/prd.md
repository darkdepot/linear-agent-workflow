# PRD Template

Use Russian for Linear PRD content.

Required sections:

```markdown
## Кратко

## Целевой оператор

## Проблема

## Целевой workflow

## Пользовательские сценарии

## Требования

## Что не входит в MVP

## Acceptance

## Связи
```

Rules:

- PRD defines WHAT.
- Do not describe implementation architecture.
- If discovery workflows were skipped, mark the document as PRD-lite and call out lower confidence.
- Acceptance must be checkable by another agent.
